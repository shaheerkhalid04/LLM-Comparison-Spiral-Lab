# LLM Comparison – Document Summarization

Compare how different Large Language Models (LLMs) summarize the same document
using the same prompt, then evaluate the results.

## What this project does

1. Reads one source document.
2. Sends the **same prompt** to five free models (GPT-OSS, Gemma,
   Nemotron, Hunyuan, Cohere) via [OpenRouter](https://openrouter.ai) —
   one free API key covers all of them.
3. Saves each model's summary to a separate file.
4. Creates a comparison table and a report so you can score and conclude.

## Project layout

```
llm-summarization-comparison/
├── documents/
│   └── source_document.txt      # the document you want summarized
├── outputs/                     # generated summaries (one file per model)
├── results/
│   ├── prompt_used.txt          # the exact prompt sent to every model
│   ├── comparison_table.md      # scores for each metric (you fill in)
│   └── report.md                # observations + conclusion (you fill in)
├── src/
│   ├── config.py                # models, prompt, and settings
│   ├── clients.py               # API calls for each provider
│   └── run_comparison.py        # main script
├── requirements.txt
├── .env.example
└── README.md
```

## Setup

1. (Recommended) create a virtual environment:

   ```
   python -m venv venv
   source venv/bin/activate      # on Windows: venv\Scripts\activate
   ```

2. Install the libraries:

   ```
   pip install -r requirements.txt
   ```

3. Add your API key:

   ```
   cp .env.example .env        # on Windows: copy .env.example .env
   ```

   Get a free key at <https://openrouter.ai/keys> (sign up, no card
   needed), then open `.env` and paste it in. The script reads `.env`
   automatically when it runs — no need to export anything.

## Running

1. Put your article, paper, or PDF text into `documents/source_document.txt`
   (a sample is already there).

2. Run:

   ```
   python src/run_comparison.py
   ```

3. Read the summaries in `outputs/`, then fill in the scores in
   `results/comparison_table.md` and write your conclusion in
   `results/report.md`.

## Evaluation metrics

Each model is scored 1 (poor) to 5 (excellent) on:

- **Summary Quality** – is it well-structured and easy to understand?
- **Accuracy** – does it correctly represent the source?
- **Conciseness** – does it drop fluff while keeping key information?
- **Hallucinations** – does it avoid adding facts not in the source?
  (higher score = fewer made-up facts)
